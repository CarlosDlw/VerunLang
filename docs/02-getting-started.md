# Getting Started

## Installation

### Prerequisites

- **Rust** (1.85+) with Cargo
- **Z3 SMT Solver** — the Z3 shared library must be available on your system

#### Installing Z3

On Debian/Ubuntu:

```bash
sudo apt install libz3-dev
```

On Arch Linux:

```bash
sudo pacman -S z3
```

On macOS:

```bash
brew install z3
```

### Building from Source

```bash
git clone https://github.com/your-org/verun.git
cd verun
cargo build --release
```

The binary will be at `target/release/verun`. Add it to your `PATH` or use `cargo install --path .`.

## Your First Spec

Create a file called `account.verun`:

```verun
state Account {
    balance: int
    frozen: bool

    invariant non_negative {
        balance >= 0
    }

    init {
        balance = 0
        frozen = false
    }

    transition deposit(amount: int) {
        where {
            amount > 0
            frozen == false
        }
        balance += amount
        ensure {
            balance == old(balance) + amount
        }
    }

    transition withdraw(amount: int) {
        where {
            amount > 0
            amount <= balance
            frozen == false
        }
        balance -= amount
        ensure {
            balance == old(balance) - amount
        }
    }

    transition freeze() {
        where { frozen == false }
        frozen = true
        ensure { frozen == true }
    }
}
```

This defines a simple bank account with one rule: the balance can never go negative.

## Verifying Your Spec

```bash
verun check account.verun
```

Output:

```
Verifying state Account...
  [PASS] init satisfies invariant 'non_negative'
  [PASS] transition 'deposit' preserves invariant 'non_negative'
  [PASS] transition 'withdraw' preserves invariant 'non_negative'
  [PASS] transition 'freeze' preserves invariant 'non_negative'
  [PASS] postcondition of 'deposit' holds
  [PASS] postcondition of 'withdraw' holds
  [PASS] postcondition of 'freeze' holds
  [PASS] transition 'deposit' is reachable
  [PASS] transition 'withdraw' is reachable
  [PASS] transition 'freeze' is reachable

10/10 checks passed
```

Every check is a mathematical proof. The SMT solver explored all possible values of `amount`, `balance`, and `frozen` and confirmed that no combination of valid inputs can make the balance go negative.

### What If Verification Fails?

Change the `withdraw` precondition to remove the balance check:

```verun
transition withdraw(amount: int) {
    where {
        amount > 0
        // removed: amount <= balance
    }
    balance -= amount
}
```

Now run `verun check` again:

```
  [FAIL] transition 'withdraw' preserves invariant 'non_negative'
         Counterexample: balance = 0, amount = 1
```

The solver found a concrete counterexample: if `balance` is 0 and someone withdraws 1, the balance becomes -1, violating the invariant. This is the kind of bug that tests might miss (especially if nobody tests the zero-balance edge case), but formal verification catches by construction.

## Generating Code

Once your spec passes verification, generate implementation code:

```bash
# Rust
verun gen account.verun -t rust

# TypeScript
verun gen account.verun -t typescript

# Solidity
verun gen account.verun -t solidity
```

The generated code includes:
- A struct/class with all state fields
- A constructor from the `init` block
- Methods for each transition with `require()`/assertion guards
- Invariant checks after every transition

To write to a file:

```bash
verun gen account.verun -t solidity -o Account.sol
```

## Scaffolding a New Spec

Start a new spec from a template:

```bash
verun init MySystem
```

This creates `mysystem.verun` in the current directory with a minimal valid spec:

```verun
// MySystem — generated by verun init

state MySystem {
    value: int

    invariant non_negative {
        value >= 0
    }

    init {
        value = 0
    }

    transition increment(amount: int) {
        where {
            amount > 0
        }
        value += amount
        ensure {
            value == old(value) + amount
        }
    }
}
```

You can specify a different output path:

```bash
verun init MySystem -o my_system.verun
```

## Formatting

Keep your specs consistently formatted:

```bash
# Format and overwrite the file
verun fmt account.verun

# Check formatting without modifying (useful for CI)
verun fmt account.verun --check
```
